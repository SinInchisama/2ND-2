오른손 좌표계로(CCW)로 모델을 만들었다고 햇을때
LHS로 게임에서 나타난다고하면(CW) 

-변환 없이 나타내면 문제점 	1. CCW보이는 면이 CW에서는 안보인다고 생각해서 그리지않음
			2. Z축의 방향이 반대라 거울에 비친거처럼 좌우가 바뀌게됨
따라서 변환을 해야한다 (포팅)
1. 정점 재정렬 CCW(P1,P2,P3) -> CW(P1,P3,P2)
2. Z 좌표 부호 변경 => 모든 Z좌표에 -1곱하기(EXPORT작업할때)

좌표 부호 변경은 필수이지만 정점 재정렬은 피할 수 있음.


렌더링 파이프라인
- 정점 처리 : 정점 버퍼의 정점에 대해 변환 등의 연산 수행
  레스터화 : 변환된 정점으로 다각형(폴리곤)을 만들고 다각형을 구성하는 픽셀들을 얻어 삼각형 내부를 채우는 프래그먼트(컬러 버퍼{화면을 구성하는 픽셀 정체가 저장되는 메모리 공간}의 픽셀을 수정하는데 필요한 데이터 모음) 생성.
  프래그먼트 처리 : 각각의 프래그먼트에 대해서 텍스쳐링 등의 작업을 거쳐 색상을 결정
  출력 병합 : 프래그먼트와 현재 컬러 버퍼 픽셀을 결합하거나 선택하여 컬러 버퍼 수정.

  프로그래밍 가능 단계 - 정점 처리와 프래그먼트 처리, 정점 프로그램과 프래그먼트 프로그램을 사용해서 정점 변환과 프래그먼트 색상 결정을 할 수 있음.
  하드 웨어로 고정된 단계 - 레스터화 및 출력 변합, 파라미터를 사용한 작동 방식 설정 가능
